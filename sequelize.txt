Migration : 

Tout comme vous utilisez des systèmes de contrôle de version tels que Git pour gérer les modifications de votre code source, vous pouvez utiliser les migrations pour suivre les modifications apportées à la base de données. Avec les migrations, vous pouvez transférer votre base de données existante dans un autre état et vice versa : ces transitions d'état sont enregistrées dans des fichiers de migration, qui décrivent comment accéder au nouvel état et comment annuler les modifications afin de revenir à l'ancien état. Vous aurez besoin de l'interface de ligne de commande Sequelize (CLI). L'interface de ligne de commande fournit un support pour les migrations et l'amorçage de projet. Une migration dans Sequelize est un fichier javascript qui exporte deux fonctions, haut et bas, qui dictent comment effectuer la migration et l'annuler. Vous définissez ces fonctions manuellement, mais vous ne les appelez pas manuellement ; ils seront appelés automatiquement par la CLI. Dans ces fonctions, vous devez simplement effectuer toutes les requêtes dont vous avez besoin, à l'aide de sequelize.query et des autres méthodes que Sequelize vous fournit. Il n'y a pas de magie supplémentaire au-delà de cela.
